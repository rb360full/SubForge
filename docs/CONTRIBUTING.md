# Contributing

Thank you for helping improve SubForge.

## Principles

- Keep changes focused
- Prefer small, reviewable pull requests
- Preserve modular boundaries
- Add tests for new behavior
- Update documentation when needed

## Setup

1. Create a virtual environment
2. Install dependencies from `requirements.txt`
3. Run linting and tests locally before opening a change

## Expectations

- Do not hardcode configuration values
- Do not merge provider-specific logic into shared core stages
- Do not add networking or scraping logic without a clear implementation target
